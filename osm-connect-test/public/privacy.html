Privacy and data handling

Why the OSM connection is required

This application exists to prove that a third party application can authenticate with Online Scout Manager and read information correctly. It cannot test that without a real OSM sign in.

What information may be read

  • Your OSM user identifier, and whether OSM returned a name and email address for you.
  • The groups and sections your OSM account can reach, and their types.
  • The permission categories and values OSM returns for each section.
  • The terms available to the selected section.
  • Where an administrator has explicitly enabled a higher-risk test, the structure of a programme, attendance, badge or member response — never the content, which is sanitised.

What is stored

  • A hashed reference to your OSM user identifier. The raw identifier is not stored.
  • Your OSM access and refresh tokens, encrypted with AES-256-GCM and readable only by the server.
  • Masked group and section identifiers, section names, section types and permission summaries.
  • Test results: stage names, statuses, timings, message codes and correlation identifiers.
  • Sanitised request records: method, destination path, query parameter names, HTTP status, content type, response size, parse result and a redacted response preview.
  • Audit records of connections, tests, exports and configuration changes.

What is never stored

Passwords, security answers, the client secret, authorisation codes, OAuth state values, cookies, full personal names, dates of birth, addresses, contact details, medical information, payment details, free-text member notes, and full unredacted request or response bodies.

How long it is stored

Diagnostic data is removed after the configured retention period. Audit records are retained for security purposes.

Who can view diagnostics

  • Testers see only their own test history and sanitised diagnostics.
  • Developers see expanded, still sanitised, request and schema diagnostics.
  • Administrators see application-wide outcomes and audit records, but cannot view any user's access token or the client secret.

Personal OSM data is not used for analytics and is not sent to any third party error-monitoring service.

How to disconnect

Use Disconnect at any time. That removes the tokens held by this application and ends your session.

How to request deletion

You can delete your test history, stored section references and OSM tokens now. Audit records are retained as required for security.
